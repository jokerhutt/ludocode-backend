## Overview

The Coins Service manages a user's coin balance — the primary in-app currency awarded for completing lessons.  
It exposes read and write operations for coin totals, ensures coin rows exist for every authenticated user,  
and applies coin deltas generated by lesson completion.

The service is intentionally simple: it maintains one integer counter per user and relies on other services  
(e.g., [Lesson Completion](lesson-completion-service.md)) to decide when coins should be awarded.

---

## Responsibilities

- Initialize a `UserCoins` record when a user logs in for the first time
- Return coin balances for one or multiple users
- Apply coin changes resulting from lesson completions
- Persist updated coin totals

---

## Boundaries & Non-Responsibilities

The Coins Service does **not**:

- compute lesson correctness
- perform streak calculations
- determine how many coins an action is worth
- run business rules for XP, multipliers, or streak rewards

Those concerns belong to the [**Lesson Completion Service**](lesson-completion-service.md), which calls this service with concrete deltas.

---

## Authentication Integration

During login, the Auth Service executes:

```
val coins = userCoinsPortForAuth.findOrCreateCoins(user.id)
```

This makes sure:
1. Every authenticated user always has a coin row
2. New users start at zero coins
3. Stale users get their coin state fetched without side effects

This is also how the streak initialization works.

## Data Models

### Persisted Entity
```
UserCoins
    userId: UUID   // primary key
    coins: Int     // the user's current balance
```

### Response DTO
```
UserCoinsResponse
    id: UUID
    coins: Int
```

### Delta Model
```
PointsDelta
    userId: UUID
    pointsDelta: Int   // + or -
```

---

## Core Operations

- `findOrCreateCoins(userId)`  
  Initializes the row if missing and returns the current coin balance

- `apply(delta)`  
  Adds or subtracts coins based on lesson results, then returns the updated balance

- `getUserCoinsList(userIds)`  
  Batched read operation for leaderboards, social feeds, etc.

---

## Public API

| Method | Path                  | Returns                   | Purpose                                 |
|------: |-----------------------|---------------------------|-----------------------------------------|
|   GET  | `/progress/coins/ids` | `List<UserCoinsResponse>` | Return coin balances for multiple users |

This API is intentionally read-only from an external perspective.  
Coin increments are performed internally via the Lesson Completion Service.

---

## Integration Points

**Inbound**
- Auth Service → initializes coins on login via `findOrCreateCoins`
- Lesson Completion Service → awards coins through `apply(delta)`

**Outbound**
- None; the Coins Service does not call other services

All coin state lives and evolves here.

---

## Future Work / Known Gaps

- Support for coin multipliers (e.g., streak or premium boosts)
- Transaction history for auditability
- Coin spend operations for future features (shop, boosts, retries)
